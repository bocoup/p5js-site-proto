  <p>A class to describe an

  <a
  href="https://developer.mozilla.org/en-US/docs/Learn/HTML/Introduction_to_HTML/Getting_started"

parent:
  description: >
    <p>Attaches this element to a parent element.</p>

    <p>For example, a <code>&lt;div&gt;&lt;/div&gt;</code> element may be used
    as a box to

    hold two pieces of text, a header and a paragraph. The

    <code>&lt;div&gt;&lt;/div&gt;</code> is the parent element of both the
    header and

    paragraph.</p>

    <p>The parameter <code>parent</code> can have one of three types.
    <code>parent</code> can be a

    string with the parent element's ID, as in

    <code>myElement.parent('container')</code>. It can also be another

    <a href="#/p5.Element">p5.Element</a> object, as in

    <code>myElement.parent(myDiv)</code>. Finally, <code>parent</code> can be an
    <code>HTMLElement</code>

    object, as in <code>myElement.parent(anotherElement)</code>.</p>

    <p>Calling <code>myElement.parent()</code> without an argument returns this
    element's

    parent.</p>

style:
  description: >
    <p>Applies a style to an element by adding a

    <a href="https://developer.mozilla.org/en-US/docs/Web/CSS/Syntax"
    target="_blank">CSS declaration</a>.</p>

    <p>The first parameter, <code>property</code>, is a string. If the name of a
    style

    property is passed, as in <code>myElement.style('color')</code>, the method
    returns

    the current value as a string or <code>null</code> if it hasn't been set. If
    a

    <code>property:style</code> string is passed, as in

    <code>myElement.style('color:deeppink')</code>, the method sets
    <code>property</code> to

    <code>value</code>.</p>

    <p>The second parameter, <code>value</code>, is optional. It sets the
    property's value.

    <code>value</code> can be a string, as in

    <code>myElement.style('color', 'deeppink')</code>, or a

    <a href="#/p5.Color">p5.Color</a> object, as in

    <code>myElement.style('color', myColor)</code>.</p>

    <code>myElement.attribute('align')</code> returns the attribute's current
    value as a

    string or <code>null</code> if it hasn't been set.</p>

    <p>The second parameter, <code>value</code>, is optional. It's a string used
    to set the

    attribute's value. For example, calling

    <code>myElement.attribute('align', 'center')</code> sets the element's
    horizontal

    alignment to <code>center</code>.</p>

size:
  description: >
    <p>Sets the element's width and height.</p>

    <p>Calling <code>myElement.size()</code> without an argument returns the
    element's size

    as an object with the properties <code>width</code> and <code>height</code>.
    For example,
     <code>{ width: 20, height: 10 }</code>.</p>
    <p>The first parameter, <code>width</code>, is optional. It's a number used
    to set the

    element's width. Calling <code>myElement.size(10)</code></p>

    <p>The second parameter, 'height<code>, is also optional. It's a number used
    to set the element's height. For example, calling </code>myElement.size(20,
    10)` sets the element's width to 20 pixels and height

    to 10 pixels.</p>

    <p>The constant <code>AUTO</code> can be used to adjust one dimension at a
    time while

    maintaining the aspect ratio, which is <code>width / height</code>. For
    example,

    consider an element that's 200 pixels wide and 100 pixels tall. Calling

    <code>myElement.size(20, AUTO)</code> sets the width to 20 pixels and height
    to 10

    pixels.</p>

    <p>Note: In the case of elements that need to load data, such as images,
    wait

    to call <code>myElement.size()</code> until after the data loads.</p>

  description: >
    <p>Sets a function to call when the user drops a file on the element.</p>

    <p>The first parameter, <code>callback</code>, is a function to call once
    the file loads.

    The callback function should have one parameter, <code>file</code>, that's a

    <a href="#/p5.File">p5.File</a> object. If the user drops multiple files on

    the element, <code>callback</code>, is called once for each file.</p>

    <p>The second parameter, <code>fxn</code>, is a function to call when the

    one or more dropped files. The callback function should have one

    parameter, <code>event</code>, that's a
