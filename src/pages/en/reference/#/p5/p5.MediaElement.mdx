volume:
  description: >
    <p>Manages the audio/video volume.</p>

    <p>Calling <code>media.volume()</code> without an argument returns the
    current volume

    as a number in the range 0 (off) to 1 (maximum).</p>

    <p>The parameter, <code>val</code>, is optional. It's a number that sets the
    volume

    from 0 (off) to 1 (maximum). For example, calling
    <code>media.volume(0.5)</code>

    sets the volume to half of its maximum.</p>
  path: ./src/pages/en/reference/#/p5.MediaElement/
speed:
  description: >
    <p>Manages the audio/video playback speed. Calling
    <code>media.speed()</code> returns

    the current speed as a number.</p>

    <p>The parameter, <code>val</code>, is optional. It's a number that sets the
    playback

    speed. 1 plays the media at normal speed, 0.5 plays it at half speed, 2

    plays it at double speed, and so on. -1 plays the media at normal speed

    in reverse.</p>

    <p>Note: Not all browsers support backward playback. Even if they do,

    playback might not be smooth.</p>
  path: ./src/pages/en/reference/#/p5.MediaElement/

addCue:
  description: >
    <p>Schedules a function to call when the audio/video reaches a specific time

    during its playback.</p>

    <p>The first parameter, <code>time</code>, is the time, in seconds, when the
    function

    should run. This value is passed to <code>callback</code> as its first
    argument.</p>

    <p>The second parameter, <code>callback</code>, is the function to call at
    the specified

    cue time.</p>

    <p>The third parameter, <code>value</code>, is optional and can be any type
    of value.

    <code>value</code> is passed to <code>callback</code>.</p>

    <p>Calling <code>media.addCue()</code> returns an ID as a string. This is
    useful for

    removing the cue later.</p>
